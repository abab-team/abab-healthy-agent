# 禁止事项

本文档列出 family-health-agent 的硬性禁止事项。任何任务、代码审查和合并都必须以此为阻断规则。

## 架构禁止事项

- 不允许重构顶层目录结构。
- 不允许提前实现后续 Phase。
- 不允许把所有 models、services、schemas 堆在根目录。
- 不允许为了赶进度破坏模块边界。
- 不允许创建与当前任务无关的脚手架、服务、模型或集成。

## 分层禁止事项

- 不允许把业务逻辑写进 API 层。
- 不允许 API 层直接访问 repository 或数据库。
- 不允许 Repository 写业务决策。
- 不允许 Service 依赖 LangGraph workflow。
- 不允许 Workflow 直接创建数据库 session。
- 不允许 Tool 绕过 service。

## Agent 与 LLM 禁止事项

- 不允许 Agent 直连数据库。
- 不允许 Agent Tools 直接 import repository 或数据库 session。
- 不允许 Tool 自己决定查询谁。
- 不允许 LLM 决定 `current_user_id`。
- 不允许 LLM 决定 `family_id`。
- 不允许 LLM 决定 `target_user_id`。
- 不允许 LLM 调用写入工具。
- 不允许在测试里依赖真实 LLM API。
- 不允许 document 工具返回真实文件路径给 LLM。

## 权限禁止事项

- 不允许绕过权限系统。
- 不允许跳过 family membership 检查访问家人数据。
- 不允许权限不足时透露数据是否存在。
- 不允许前端传什么 target_user_id 后端就信什么。
- 不允许用 LLM 输出替代后端权限判断。

## AI 草稿与写入禁止事项

- 不允许未确认的 AI 草稿直接入库为正式记录。
- 不允许没有用户确认就写入健康档案。
- 不允许 AI 自行生成正式医疗事实。
- 不允许把草稿状态伪装成正式记录。
- 不允许写入失败时吞掉错误。

## 医疗安全禁止事项

- 不允许实现医学诊断。
- 不允许实现处方建议。
- 不允许实现药物剂量建议。
- 不允许输出开药、停药、换药建议。
- 不允许保证用户“没事”。
- 不允许夸大风险制造恐慌。
- 不允许把“系统无记录”说成“现实没有问题”。
- 不允许让日报、就医摘要、知识库问答生成诊断或治疗方案。

## 数据与隐私禁止事项

- 不允许把密钥、Token、真实隐私数据写入仓库。
- 不允许日志记录完整敏感健康内容，后续必须做脱敏。
- 不允许未审计地访问敏感家庭健康数据。
- 不允许共享链接不可撤销。
